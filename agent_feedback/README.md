# Agent Feedback Loop

This folder is a lightweight, repo-local feedback mechanism to keep Nanoclaw merge-ready and to make iterative improvements explicit.

## How to use

- Write new feedback into `latest.md` after each milestone.
- Capture completed work as short task logs (one file per task) so we preserve intent, decisions, and follow-ups.
- Keep entries actionable: what changed, why, and what to do next.

## Files

- `latest.md`: current feedback / next actions
- `gas-protector.md`: task log for Gas Protector integration
- `profit-logic.md`: task log for profit-taking logic improvements
- `cleanup-tests.md`: task log for cleanup + unit test stabilization
